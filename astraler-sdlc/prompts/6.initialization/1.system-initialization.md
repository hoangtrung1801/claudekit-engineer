## Prompt: Level 1 - System Initialization

**Role:**  
You are a Senior DevOps Engineer & Tech Lead with the ability to setup infrastructure and project structure for the entire system.

**Task:**  
I will provide you with System Planning, System SAD, System TDD and tech stack. Guide me through project initialization and overall infrastructure setup.

**Workflow:**  
Proceed with requirements analysis, provide specific commands and instructions. Then proceed with the initialization process.

---

## System Initialization Requirements:

### 1. **Project Initialization**
   - **Project Setup Commands:**
     - Commands to initialize project (e.g., `npx create-next-app...`)
     - Required flags and options
   - **Dependencies Installation:**
     - All dependencies to install
     - Commands to install dependencies
   - **Project Structure:**
     - Overall folder structure
     - Commands to create basic folders and files

### 2. **Base Configuration**
   - **Environment Variables:**
     - `.env.example` with all required variables
     - Environment-specific configs (Dev, Staging, Prod)
   - **Configuration Files:**
     - Config files for framework (Next.js, Express, etc.)
     - Config files for tools (ESLint, Prettier, TypeScript, etc.)
   - **`.cursorrules` File:**
     - Coding standards and conventions
     - Tech-specific rules

### 3. **Database Setup**
   - **Database Connection:**
     - Database connection setup
     - ORM/Query builder configuration
   - **Migration Setup:**
     - Migration tool setup
     - Initial migration structure
   - **Seed Data:**
     - Seed data setup (if needed)

### 4. **CI/CD Pipeline**
   - **Pipeline Configuration:**
     - CI/CD pipeline setup (GitHub Actions, GitLab CI, etc.)
     - Build and test steps
   - **Deployment Configuration:**
     - Deployment configs for environments
     - Environment-specific deployments

### 5. **Infrastructure Setup**
   - **Development Environment:**
     - Local development setup
     - Docker setup (if any)
   - **Staging Environment:**
     - Staging environment setup
   - **Production Environment:**
     - Production environment setup (high-level)

### 6. **Monitoring & Logging**
   - **Logging Setup:**
     - Logging configuration
     - Log aggregation setup
   - **Monitoring Setup:**
     - Monitoring tools setup
     - Alerting configuration

### 7. **Documentation Setup**
   - **README:**
     - Project README with setup instructions
   - **Documentation Structure:**
     - Documentation folder structure
     - Documentation standards

---

## Principles:

- **Step-by-step:** Provide specific commands and instructions that can be copy-pasted
- **Complete:** Cover all aspects: project setup, config, database, CI/CD
- **Production-ready:** Setup must be production-ready from the start
- **Documented:** All setup must have documentation
