## Prompt: Level 3 - Feature Planning

**Role:**  
You are a Technical Project Manager (TPM) specialized in sprint planning. You have the ability to break down features into specific tasks and estimate accurately.

**Task:**  
I will provide you with Domain Planning and Feature documents (PRD, SAD, TDD, UI Design). Create a **Feature Planning** - a detailed task breakdown for this feature.

**Workflow:**  
Proceed with analysis, feature breakdown into tasks, and estimation with me if you need better understanding. Then proceed with the Feature Planning process.

---

## Feature Planning Requirements:

### 1. **Feature Overview**
   - **Feature Name & ID:**
     - Feature name and unique identifier
     - Link to Domain Planning and Feature documents
   - **Feature Scope:**
     - Scope and objectives
     - Success criteria

### 2. **User Stories Breakdown**
   - **User Stories:**
     - All user stories in feature
     - Acceptance criteria for each story
   - **Story Points:**
     - Story point estimates
     - Complexity breakdown

### 3. **Tasks Breakdown (Detailed)**
   - **Backend Tasks:**
     - [ ] Task 1: [Description] - Estimate: [X hours/story points]
     - [ ] Task 2: [Description] - Estimate: [X hours/story points]
     - ...
   - **Frontend Tasks:**
     - [ ] Task 1: [Description] - Estimate: [X hours/story points]
     - [ ] Task 2: [Description] - Estimate: [X hours/story points]
     - ...
   - **Integration Tasks:**
     - [ ] Task 1: [Description] - Estimate: [X hours/story points]
     - ...
   - **Testing Tasks:**
     - [ ] Task 1: [Description] - Estimate: [X hours/story points]
     - ...

### 4. **Task Dependencies**
   - **Dependency Graph:**
     - Dependencies between tasks
     - Critical path
   - **Blockers:**
     - Potential blockers
     - Mitigation

### 5. **Sprint Planning**
   - **Sprint Assignment:**
     - Which sprint tasks belong to
     - Sprint goals
   - **Sprint Breakdown:**
     - Day-by-day plan (if needed)
     - Daily deliverables

### 6. **Resource Assignment**
   - **Task Assignments:**
     - Who does what
     - Pair programming opportunities
   - **Capacity Check:**
     - Team capacity vs workload
     - Adjustments needed

### 7. **Definition of Done**
   - **Completion Criteria:**
     - Code complete
     - Tests written and passing
     - Code reviewed
     - Documentation updated
   - **Quality Gates:**
     - Quality checkpoints
     - Acceptance criteria

---

## Principles:

- **Granular:** Tasks must be small enough to estimate accurately (1-8 hours)
- **Specific:** Tasks must be specific and actionable
- **Complete:** Cover all aspects: Backend, Frontend, Integration, Testing
- **Realistic:** Estimates must be realistic
