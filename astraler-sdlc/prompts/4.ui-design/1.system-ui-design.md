## Prompt: Level 1 - System UI Design

**Role:**  
You are a Senior UI/UX Designer & Frontend Architect with the ability to design overall Design System for the entire system. You understand Color Psychology and modern design principles.

**Task:**  
I will provide you with System PRD, System SAD, System TDD and brand guidelines. Create a **System UI Design** - an overall Design System for the entire system.

**Workflow:**  
Proceed with analysis, brainstorming, and breakdown with me if you need better understanding. Then proceed with the System UI Design process.

---

## System UI Design Requirements:

### 1. **Design Philosophy**
   - **Visual Identity:**
     - Describe the vibe the application conveys (e.g., Trustworthy, Clean, Playful, Professional)
     - Brand personality and tone of voice
   - **Design Principles:**
     - Core design principles (e.g., Simplicity, Consistency, Accessibility)
     - Design approach (e.g., Modern Minimalist, Brutalism, Glassmorphism)
   - **Spacing Philosophy:**
     - Dominant spacing: Spacious or Compact?
     - Spacing scale and rhythm
   - **Border Radius Philosophy:**
     - Dominant corners: Rounded (Playful) or Square (Serious)?
     - Radius scale

### 2. **Color Palette**
   - **Primary Colors:**
     - Main brand color with HEX code and Tailwind variables
     - Primary variants (light, dark, hover states)
   - **Secondary Colors:**
     - Supporting colors with variants
   - **Neutral Colors:**
     - Background/Foreground colors for Light/Dark mode
     - Gray scale with shades
   - **Semantic Colors:**
     - Success (Green), Warning (Yellow), Destructive (Red), Info (Blue)
     - Each color has variants for different states
   - **Accessibility:**
     - Ensure contrast ratios meet WCAG AA standards
     - Color combinations guidelines

### 3. **Typography System**
   - **Font Families:**
     - Font for Headings (Display font)
     - Font for Body text (Readable font)
     - Font for Code/Monospace (if needed)
   - **Type Scale:**
     - Font sizes from smallest to largest
     - Line heights and letter spacing
     - Font weights (Regular, Medium, Semibold, Bold)
   - **Typography Hierarchy:**
     - Heading levels (H1-H6) with styles
     - Body text styles
     - Caption, Label, Small text styles

### 4. **Component Library (Overview)**
   - **Core Components:**
     - Buttons: Variants and usage rules
     - Inputs: Text, Number, Select, Checkbox, Radio
     - Cards: Variants and usage
     - Modals/Dialogs: Usage patterns
   - **Navigation Components:**
     - Header/Navbar patterns
     - Sidebar patterns
     - Breadcrumbs, Tabs, Pagination
   - **Feedback Components:**
     - Alerts, Toasts, Notifications
     - Loading states (Spinner, Skeleton)
     - Empty states and Error states
   - **Data Display Components:**
     - Tables, Lists, Grids
     - Charts and Data visualization (if any)

### 5. **Layout System**
   - **Container System:**
     - Max widths for different breakpoints
     - Container padding and margins
   - **Grid System:**
     - Breakpoints (Mobile, Tablet, Desktop, Large Desktop)
     - Grid columns for each breakpoint
     - Gap system
   - **Spacing System:**
     - Spacing scale (4px, 8px, 12px, 16px, 24px, 32px, etc.)
     - Usage guidelines
   - **Layout Patterns:**
     - Sidebar layout
     - Topbar layout
     - Full-width layout
     - Centered content layout

### 6. **Design Tokens**
   - **Spacing Tokens:**
     - All standardized spacing values
   - **Color Tokens:**
     - All color values as tokens
   - **Typography Tokens:**
     - Font sizes, weights, line heights
   - **Shadow Tokens:**
     - Elevation system (shadows for depth)
   - **Border Tokens:**
     - Border widths and styles
   - **Animation Tokens:**
     - Duration, easing functions
     - Transition patterns

### 7. **Accessibility Guidelines**
   - **Color Contrast:**
     - Minimum contrast ratios
     - Color-blind friendly guidelines
   - **Keyboard Navigation:**
     - Focus states and focus management
     - Tab order guidelines
   - **Screen Reader Support:**
     - ARIA labels and roles
     - Semantic HTML guidelines
   - **Touch Targets:**
     - Minimum touch target sizes
     - Spacing between interactive elements

### 8. **Responsive Design Strategy**
   - **Breakpoints:**
     - Mobile, Tablet, Desktop breakpoints
     - Breakpoint usage guidelines
   - **Mobile-First Approach:**
     - Mobile design principles
     - Progressive enhancement strategy
   - **Responsive Patterns:**
     - Navigation patterns for mobile
     - Content stacking patterns
     - Image and media handling

### 9. **Dark Mode Strategy**
   - **Color Mapping:**
     - Light mode ‚Üí Dark mode color mapping
     - Dark mode specific colors
   - **Implementation:**
     - Theme switching mechanism
     - Persistence strategy

### 10. **Animation & Micro-interactions**
    - **Animation Principles:**
      - When to animate and when not to
      - Animation duration guidelines
    - **Micro-interactions:**
      - Hover states
      - Click/tap feedback
      - Loading animations
      - Transition patterns

### 11. **Icon System**
    - **Icon Library:**
      - Icon library choice (e.g., Lucide, Heroicons)
      - Icon style guidelines
    - **Icon Usage:**
      - Icon sizes
      - Icon placement guidelines
      - Decorative vs Functional icons

### 12. **Domain Breakdown (UI View)**
    - **UI Domains:**
      - Divide UI into domains/modules
      - Brief description of UI characteristics for each domain
    - **Domain-Specific Patterns:**
      - Patterns specific to each domain (if any)
    - **Consistency Guidelines:**
      - Ensure consistency between domains

---

## Principles:

- **System-wide focus:** Focus on overall design system, don't go into detail for each screen
- **Scalability:** Design system must be scalable and maintainable
- **Consistency:** Ensure consistency across the entire system
- **Accessibility:** Follow accessibility standards
- **Practical:** Design tokens and components must be implementable
