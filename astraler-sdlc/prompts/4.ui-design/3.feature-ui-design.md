## Prompt: Level 3 - Feature UI Design

**Role:**  
You are a Senior UI/UX Designer specialized in implementation. You have the ability to design in detail a specific feature/screen with enough information for developers to code immediately.

**Task:**  
I will provide you with:
- Domain UI Design (already available from Level 2)
- Feature PRD (already available from BA)
- Feature TDD (already available)
- User stories and acceptance criteria

Create a **Feature UI Design** - a detailed UI design document for this feature/screen, sufficient for developers to code immediately.

**Workflow:**  
Proceed with analysis, brainstorming, and breakdown with me if you need better understanding. Then proceed with the Feature UI Design process.

---

## Feature UI Design Requirements:

### 1. **Feature UI Overview**
   - **Feature Name & ID:**
     - Feature name and unique identifier
     - Link to Domain UI Design, System UI Design and Feature PRD/TDD
   - **Feature Description:**
     - Detailed description of this feature's UI
     - User goals and UI objectives
   - **Screen List:**
     - List of screens in feature
     - Screen relationships and flow

### 2. **Wireframe (Detailed)**
   - **Page Structure:**
     - Detailed page structure as tree or text description
     - Layout structure with sections
   - **Component Breakdown:**
     - All components on page
     - Component hierarchy
     - Component placement and spacing
   - **Content Structure:**
     - Content hierarchy
     - Information architecture

### 3. **Component Specifications**
   - **Component List:**
     - All components required for feature
     - Component names and types
   - **Component Props:**
     - Props for each component
     - Component states and variants
   - **Component Styling:**
     - Styling details (colors, spacing, typography)
     - Responsive behavior
   - **Component Interactions:**
     - Interaction states (hover, active, disabled)
     - Click/tap behaviors

### 4. **State Management Design**
   - **UI States:**
     - All UI states (Empty, Loading, Success, Error, etc.)
     - State transitions
   - **Data States:**
     - Data loading states
     - Data error states
     - Data empty states
   - **Form States:**
     - Form validation states
     - Form submission states
   - **User Interaction States:**
     - Selected, expanded, collapsed states
     - Multi-select states

### 5. **User Flow (UI Perspective)**
   - **Primary Flow:**
     - Detailed UI flow for happy path
     - Screen transitions
   - **Alternative Flows:**
     - UI flows for alternative paths
     - Error recovery flows
   - **Edge Case Flows:**
     - UI flows for edge cases
     - Empty state flows

### 6. **Responsive Design (Detailed)**
   - **Mobile Design:**
     - Detailed mobile layout
     - Mobile-specific components and interactions
     - Mobile navigation
   - **Tablet Design:**
     - Tablet layout
     - Tablet-specific adaptations
   - **Desktop Design:**
     - Desktop layout
     - Desktop-specific features
   - **Breakpoint Specifications:**
     - Exact breakpoints for feature
     - Layout changes at each breakpoint

### 7. **Interaction Design**
   - **Micro-interactions:**
     - Hover effects
     - Click/tap feedback
     - Loading animations
     - Transition animations
   - **Gesture Support:**
     - Swipe gestures (mobile)
     - Pinch to zoom (if any)
     - Other gestures
   - **Keyboard Navigation:**
     - Tab order
     - Keyboard shortcuts
     - Focus management

### 8. **Accessibility Specifications**
   - **ARIA Labels:**
     - ARIA labels for all interactive elements
     - ARIA roles and properties
   - **Keyboard Navigation:**
     - Keyboard navigation flow
     - Focus indicators
   - **Screen Reader Support:**
     - Screen reader announcements
     - Semantic HTML structure
   - **Color Contrast:**
     - Color contrast verification
     - Alternative indicators (not just color-dependent)

### 9. **Content Specifications**
   - **Copy & Messaging:**
     - All text content on page
     - Error messages
     - Success messages
     - Empty state messages
   - **Icon Usage:**
     - Required icons
     - Icon placement and sizes
   - **Image Specifications:**
     - Image requirements (sizes, formats)
     - Image placeholders
     - Alt text

### 10. **Performance Considerations**
    - **Loading Strategy:**
      - Lazy loading requirements
      - Progressive loading
      - Skeleton screens
    - **Optimization:**
      - Image optimization requirements
      - Code splitting considerations

### 11. **Implementation Checklist**
    - **Components to Create:**
      - [ ] Component 1
      - [ ] Component 2
      - ...
    - **Pages to Create:**
      - [ ] Page 1
      - [ ] Page 2
      - ...
    - **States to Implement:**
      - [ ] Empty state
      - [ ] Loading state
      - [ ] Error state
      - ...
    - **Interactions to Implement:**
      - [ ] Hover effects
      - [ ] Click handlers
      - [ ] Animations
      - ...

### 12. **Design Handoff Specifications**
    - **Design Tokens:**
      - Colors, spacing, typography tokens used
    - **Component Library:**
      - Components from System UI Design used
      - Custom components needed
    - **Assets Needed:**
      - Images, icons, illustrations
    - **Developer Notes:**
      - Special considerations
      - Implementation hints

---

## Principles:

- **Implementation-ready:** Detailed enough for developers to code immediately without asking many questions
- **Specific:** Specific about components, states, interactions - not vague
- **Complete:** Cover all aspects: layout, components, states, interactions, responsive
- **Traceable:** Clear link to Domain UI Design, System UI Design and Feature PRD/TDD
- **Practical:** Focus on what's needed to implement this feature
