## ğŸ“Š When to Use Each Level?

### **Level 1 (System):**
- Starting a new project
- Need overall system overview
- Breaking down project into domains/modules

### **Level 2 (Domain):**
- After having System level
- Domain has > 5 complex features
- Domain can be developed independently

### **Level 3 (Feature):**
- After having Domain level
- Feature needs immediate development
- Feature has complex business logic

## ğŸ’¡ Best Practices

1. **Always start from System level** - Don't skip System PRD/SAD/TDD
2. **Iterative approach** - Can return to System level to refine
3. **Consistency** - Ensure Domain and Feature levels align with System level
4. **Traceability** - Each document must reference related documents
5. **Parallel work** - After System levels, can work on multiple Domain levels in parallel
6. **Review points** - Review after each System level before continuing
